# 可复用代码推荐

## 工作流（三级降级链，严格按序，每级有结果即终止）

### 第一级：recommend_component（唯一首选）

1. 用户询问可复用组件/函数/util/工具 → 必须先调用 `recommend_component`
2. 返回 recommended != null → 将工具返回文本**原样输出**，**完全停止**，不得继续调用任何工具
3. 返回 recommended = null → 进入第二级

### 第二级：search_symbols（仅在第一级无结果时）

4. 调用 `search_symbols`（semantic=true，传入原始 query）
5. 返回非空结果 → 取第一条，按固定回复结构输出，**完全停止**
6. 返回空结果 → 进入第三级

### 第三级：输出无结果模板

7. 直接输出无结果固定模板，**完全停止**
8. 禁止进行 grep、read file、file search 等文件系统操作

## 硬性约束

- 不得跳过第一级直接调用 `search_symbols`
- `recommend_component` 返回后，必须将工具返回文本原样输出，不得改写为散文或追加候选
- 任何推荐结果在回答结尾都必须展示：
    -   1. 采纳推荐
    -   2. 取消
- 如果用户继续追问详情（例如 props、最小接入方式、为什么推荐它），可以补充说明，但**回答结尾必须再次原样展示这两个选项**，不得省略
- 用户选择“采纳推荐”后，必须调用 `inc_usage`

## 回复结构

有结果时：

- **首选：** `<代码块名> — <文件路径>`
- **使用范围：** callers 或“新增”
- **副作用：** sideEffects 或“无”
- **理由：** 1～3 条要点
- **其他候选：** 简要列出及取舍（同步标注副作用）
- **用法提示：** 最小集成示例
- **是否采纳：**
    -   1. 采纳推荐
    -   2. 取消

无结果时：

- **首选：** 未找到已有实现
- **使用范围：** 无
- **副作用：** 无
- **理由：** 说明未命中原因
- **其他候选：** 无
- **用法提示：** 给出最小可复用实现方向
- **是否采纳：**
    -   1. 让我新建一个最小可复用实现
    -   2. 取消

## 使用反馈

当用户选择“采纳推荐”后，必须调用：

```
inc_usage({ symbolId: <选中的代码块 id> })
```

symbolId 从推荐结果中的 id 字段获取。
